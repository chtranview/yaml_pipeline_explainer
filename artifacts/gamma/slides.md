## M2｜壞例子 vs 好例子


## Prompt → Skill 設計對照

## 壞例子：一次性 Prompt（Bad Example）
- **問題在哪？**
- 沒有明確 Trigger
- 沒有定義 Input
- Instruction 太模糊
- Output 格式不固定

## 好例子：可重用 Skill（Good Example）
- **Skill Name**
- **Trigger**
- **Input**
- article_text
- **Instruction**
- **Output Contract**
- **Constraints**

## 關鍵差異
- **壞 Prompt 的問題**
- **好 Skill 的特徵**

## 從 Prompt 到 Skill：設計可重用的 AI 任務單元


- **所屬天次 / 順序**：Day 1 / Module 2
- **預計時長**：90 分鐘
- **難度**：🔶 中階

## 學習目標
- 能夠將單次 Prompt 重構為可重複使用的 Skill
- 學會撰寫包含 trigger、instruction、output 的任務規格
- 建立一份可直接操作的 AI Skill 定義稿

## 核心概念
- **1. Prompt vs Skill**
- Prompt：「幫我整理這篇文章」
- Skill：「當我貼入文章時，請輸出摘要、重點、可行動項目，格式固定」
- **2. Trigger**
- **3. Instruction**
- **4. Output Contract**

## AI Workflow MVP（展示級原型）


- 一些學員焦慮
- 一些學員失焦
- 一些學員瘋狂加功能，最後做不完

## 建議替換文案（補進 M4 handout / deck）
- **`artifacts/handouts/M4_positioning_patch.md`**

## 本模組最終成果不是：
- 正式上線產品
- 完整 App
- 真正串 API 的自動化系統

## 本模組最終成果是：
- 你可以清楚說明它解決什麼問題
- 你可以展示它的流程怎麼跑
- 你可以證明它有明確輸出價值
- 你可以在 3 分鐘內讓別人理解它為什麼值得做

## 這代表什麼？
- **你今天要完成的是：**
- 一個 **可展示**
- 可說明
- 可驗證
- 可帶回去延伸的 AI 工作流原型
- **而不是：**

## 評估標準會看的是：
- 流程是否清楚
- 模組是否合理
- 成果是否可驗證
- Demo 是否能讓別人看懂價值
