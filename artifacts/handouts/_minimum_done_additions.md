# Minimum Done 補充區塊

---

## M1｜System Thinking
### Minimum Done（最低完成線）

本模組至少需完成以下內容，才算過關：

- 至少拆出 **4 個模組**
- 每個模組都要寫出：
  - Input（輸入）
  - Output（輸出）
- 明確標出至少 **1 個依賴關係**
- 指出至少 **1 個最適合交給 AI 的模組**

### 快速自檢
如果別人看你的圖後，能回答下面 3 題，就算合格：
1. 這個流程在做什麼？
2. 每個模組在處理哪一段？
3. 哪一段最值得 AI 化？

## M2｜Prompt Skill Design
### Minimum Done（最低完成線）

本模組至少需完成以下內容，才算過關：

- 完成 1 份 Skill 草稿
- 至少包含以下 4 個核心欄位：
  - Skill Name
  - Trigger
  - Instruction
  - Output Contract
- Input 欄位至少定義 1 項
- Instruction 至少拆成 **3 個可執行步驟**
- Output 至少定義 **2 個固定輸出項目**

### 快速自檢
如果另一位同學拿到你的 Skill，能直接理解：
- 什麼時候用
- 要丟什麼資料
- 最後會拿到什麼

就算合格。

## M3｜YAML Course Architecture
### Minimum Done（最低完成線）

本模組至少需完成以下內容，才算過關：

- 完成 1 份 YAML 模組草稿
- 至少包含以下 4 個核心欄位：
  - `module_id`
  - `inputs`
  - `steps`
  - `outputs`
- `steps` 至少寫出 **3 個步驟**
- 至少寫出 **1 條 validation rule**
- 至少寫出 **1 條 error handling**

### 快速自檢
如果別人只看你的 YAML，也能說出：
- 這模組在做什麼
- 它吃什麼資料
- 它會產出什麼

就算合格。

---

## M4｜Workshop MVP Build
### Minimum Done（最低完成線）

本模組至少需完成以下內容，才算過關：

- 串出至少 **2 個模組**
- 每個模組都標出：
  - Input
  - Output
- 定義至少 **3 條 Success Criteria**
- 完成 1 份 **3 分鐘 Demo Script**
- 清楚說出：
  - 解決什麼問題
  - 流程怎麼跑
  - 產出長什麼樣

### 快速自檢
如果你能在 3 分鐘內讓別人理解這個 MVP 的價值，就算合格。
