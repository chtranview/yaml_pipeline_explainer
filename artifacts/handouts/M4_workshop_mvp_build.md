# Workshop MVP：組裝一個可運作的 AI 系統原型

- **所屬天次 / 順序**：Day 2 / Module 4
- **預計時長**：90 分鐘
- **難度**：🔴 進階

## 學習目標
- 能夠將多個模組串成一個可運作的 AI 工作流
- 學會定義 MVP 範圍與完成標準
- 完成一個可展示、可驗證的 AI 系統原型

## 核心概念

### 1. MVP
**一句話定義**：只做最小但足以展示價值的版本。  
**比喻**：先做會動的腳踏車，不是直接做高配版汽車。  
**例子**：先讓使用者能輸入需求並產出 outline，不急著做登入、權限、資料庫。

### 2. Flow Orchestration
**一句話定義**：讓多個模組按照正確順序合作。  
**比喻**：像指揮樂團，不是每個樂手都亂吹。  
**例子**：`outline → handout → sdd → verify`

### 3. Success Criteria
**一句話定義**：事先定義「做到哪裡才算完成」。  
**比喻**：像比賽規則，不然大家都會說自己有完成。  
**例子**：輸入課程主題後，能自動產出 4 模組 outline，且每模組都有 takeaway。

### 4. Demo Readiness
**一句話定義**：把成果整理成可以在別人面前展示的狀態。  
**比喻**：不是你自己知道能跑，而是別人一看就知道它有價值。  
**例子**：準備一段 3 分鐘 Demo 腳本，清楚展示輸入、處理、輸出。
