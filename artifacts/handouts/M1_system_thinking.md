# 模組化思維：把 AI 使用流程拆成系統

- **所屬天次 / 順序**：Day 1 / Module 1
- **預計時長**：90 分鐘
- **難度**：⬜ 基礎

## 學習目標
- 能夠辨識 AI 工作流程中的輸入、處理與輸出
- 學會將日常工作任務拆成可重用模組
- 建立一張具依賴關係的 AI 系統草圖

## 核心概念

### 1. 模組化設計
**一句話定義**：把複雜流程拆成可單獨理解、替換與改進的小單元。  
**比喻**：像積木，不是一次做完整棟房子，而是先做可拼接的零件。  
**例子**：把「做課程」拆成：需求蒐集 → 大綱生成 → 講義生成 → 驗證 → 打包。

### 2. I/O 邊界
**一句話定義**：定義每個模組「吃進什麼、吐出什麼」。  
**比喻**：像咖啡機，你只需要知道放入咖啡豆和水，最後會得到咖啡。  
**例子**：  
- 輸入：課程主題、受眾、天數  
- 輸出：課程大綱 YAML

### 3. 依賴關係
**一句話定義**：有些模組必須先完成，後面的模組才能運作。  
**比喻**：你不能先裝屋頂，再蓋地基。  
**例子**：沒有 `outline.yaml`，就無法生成 handout。

### 4. 流程抽象化
**一句話定義**：把你平常「怎麼做」提升成「可以被重複執行的方法」。  
**比喻**：從「我會煮這道菜」變成「我寫出這道菜的食譜」。  
**例子**：把「我都用感覺做簡報」改成「先列受眾→再定主訊息→再展開內容」。
