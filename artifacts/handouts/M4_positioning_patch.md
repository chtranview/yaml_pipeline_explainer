# 3) M4 成果定位修正（非常重要）

這是整套課最需要「講清楚」的地方。

你原本的 M4 容易讓人以為：

> 今天最後要做出真正可部署的 App

這個期待不準，而且會造成：

- 一些學員焦慮
- 一些學員失焦
- 一些學員瘋狂加功能，最後做不完

所以要把 M4 的成果名稱明確收斂成：

> **AI Workflow MVP（展示級原型）**

不是假裝做很大，而是**做出可清楚展示價值的最小版本**。

---

## 建議替換文案（補進 M4 handout / deck）

### `artifacts/handouts/M4_positioning_patch.md`

```md
# M4 成果定位修正

## 本模組最終成果不是：
- 正式上線產品
- 完整 App
- 真正串 API 的自動化系統

## 本模組最終成果是：
# AI Workflow MVP（展示級原型）

也就是：

- 你可以清楚說明它解決什麼問題
- 你可以展示它的流程怎麼跑
- 你可以證明它有明確輸出價值
- 你可以在 3 分鐘內讓別人理解它為什麼值得做

---

## 這代表什麼？

### 你今天要完成的是：
- 一個 **可展示**
- 可說明
- 可驗證
- 可帶回去延伸的 AI 工作流原型

### 而不是：
- 一個工程完整產品
- 一個正式部署系統

---

## 評估標準會看的是：
- 流程是否清楚
- 模組是否合理
- 成果是否可驗證
- Demo 是否能讓別人看懂價值
