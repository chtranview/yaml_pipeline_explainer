# 課後落地建議｜Workshop 之後 7 天內怎麼做？

## 不要一離開教室，整套東西就留在筆記裡。
請你在 7 天內，至少完成下面其中 1 件事。

---

# Day 1–2：選一個高頻任務
請先挑一個你工作中最常重複、最值得標準化的任務，例如：

- 寫會議摘要
- 做課程大綱
- 整理訪談內容
- 產出提案草稿
- 整理需求與行動項目

### 原則
不要選最複雜的，先選 **最常做的**。

---

# Day 3–4：先做半自動，不要急著全自動

## 很多人一開始就想：
- 串 API
- 做 App
- 接資料庫
- 做完整流程

### 這通常太快了。

你應該先做到：
- Prompt 穩定
- Skill 可重用
- YAML 可理解
- 流程可展示

### 先讓流程穩，再談技術升級。

---

# Day 5：找一位同事做「交接測試」

把你今天做的 Skill / YAML / MVP 給另一位同事看，測試他能不能回答：

1. 這套東西解決什麼問題？
2. 什麼時候該用？
3. 輸入什麼資料？
4. 最後會得到什麼？

### 如果對方看不懂
不是他不夠懂，而是你的規格還不夠清楚。

---

# Day 6–7：做一次真實工作情境試跑

請拿一個真實案例跑一次，不要只測理想狀況。

### 你要觀察：
- 哪個步驟最容易卡住？
- 哪個輸出最不穩？
- 哪個欄位最容易忘？
- 哪段流程最值得優化？

---

# 下一階段升級路徑

## 如果你想把這套東西做得更完整，可以往下走：

### Level 1｜可重用
- Skill 固定可用
- YAML 清楚可交接

### Level 2｜可展示
- 有 MVP 流程
- 有 Demo Script
- 有 Success Criteria

### Level 3｜可部署
- 串 API
- 串前端
- 做簡單內部工具

---

# 最後提醒

## 先讓它能用，再讓它變強。
你不需要一週內做出完整產品。  
你需要的是：

> 找到一個真正值得落地的 AI 工作流，
> 然後把它做穩。
